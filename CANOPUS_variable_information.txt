Harness Variables:
=======================

CANOPUS_GROUP_SUITEFILE            : [ Group testing file after processing command line ]
CANOPUS_ARGUMENT_FILE              : [ Argument command line stored to file after --input-file processing ]
CANOPUS_INPUT_FILE                 : [ User filename detailing tests to execute ]
CANOPUS_OUTPUT_FORMATS             : [ User selected output formats for test result dictation ]
CANOPUS_TAG                        : [ User specified tags for test selection ]
CANOPUS_USE_TAGS                   : [ Should tags be used to select tests to run ]
CANOPUS_TAG_ALIAS                  : [ Tag alias to interpret when scanning tests for tag fields ]
CANOPUS_PROGRAM_VERSION            : [ Harness version ]
CANOPUS_PROGRAM_VERSION_BUILD      : [ Harness build version ]
CANOPUS_PROGRAM_BUILD_DATE         : [ Harness build date ]
CANOPUS_DISPLAY_NEWLINE_MARKER     : [ Newline marker ]
CANOPUS_LOGFILE                    : [ Output logfile from test harness ]
CANOPUS_START_TIME                 : [ Starting time of running test harness ]
CANOPUS_END_TIME                   : [ Ending time of running test harness *** ]
CANOPUS_PROGRAM_OPTIONS            : [ Program options for the test harness ]
CANOPUS_TEST_FILE_EXCLUDE_PATTERNS : [ Test file exclusion patterns ]
CANOPUS_TEST_DIR_EXCLUDE_PATTERNS  : [ Test directory exclusion patterns ]
CANOPUS_PREMATURE_EXIT_MSG         : [ Message response from a premature exit ]
CANOPUS_CURRENT_TEST_LOCATION      : [ Current location whilst processing a subsystem in the test harness ]
CANOPUS_DBL_DIVIDER                : [ Divider ]
CANOPUS_PREFIX_START               : [ Prefix stating 'START' ]
CANOPUS_TEST_SUBSYSTEM             : []
CANOPUS_TEST_SUMMARY_PATH          : [ Base path for all output generated when running the test harness ]
CANOPUS_PROGRAM_OPTIONS            : [ All known option types known to the test harness ]
CANOPUS_MULTI_OPTIONS              : [ Long multi-assignment options known to the test harness ]
CANOPUS_SINGLE_OPTIONS             : [ Long single assignment options known to the test harness ]
CANOPUS_SHORT_OPTION_MATCHES       : [ A one-to-one mapping of short to long options known to the test harness ]
CANOPUS_SHORT_MULTI_OPTIONS        : [ Short multi-assignment options known to the test harness ]
CANOPUS_SHORT_SINGLE_OPTIONS       : [ Short single assignment options known to the test harness ]
CANOPUS_HOTSPOT                    : [ Channel used to record steps and overhead ]
CANOPUS_RECORD_STEPS               : [ Allow recording of steps to help hotspot analysis -- used in conjunction with another program for analysis ]
CANOPUS_RECORD_OVERHEAD            : [ Allow recording of "groups of code" to assign to static or dynamic overhead to help hotspot analysis -- used in conjunction with another program for analysis ]
CANOPUS_STARTUP_DIR                : [ Directory from which the test harness exists ]
CANOPUS_LAUNCH_DIR                 : [ Original directory which initiated the test harness ]
CANOPUS_PREMATURE_EXIT             : [ 0/1 -- Setting to determine if premature exit occured ]
CANOPUS_TEST_FILE_EXCLUDE_PATTERNS : [ Test file exclusion patterns set by the user to be honored by the test harness ]
CANOPUS_TEST_DIR_EXCLUDE_PATTERNS  : [ Directory exclusion patterns set by the user to be honored by the test harness ]
CANOPUS_SUBSYSTEM_MAP              : [ Map data representing the statistics of testing a particular subsystem ]
CANOPUS_OVERALL_MAP                : [ Map data representing the statistics of all subsystem testing ]
CANOPUS_TEST_SUBSYSTEM             : [ Current name of the subsystem under test ]
CANOPUS_TEST_PREVIOUS_SUBSYSTEM    : [ Previous name of the subsystem completed testing (if not the first subsystem) ]
CANOPUS_EMAIL_FILE                 : [ Name of the email file used to record results when the EMAIL output option is enabled ]
CANOPUS_JUNIT_XML_FILE             : [ Name of the junit xml file used to record results when the JUNIT_XML output option is enabled ]
CANOPUS_CURRENT_TEST_LOCATION      : [ Directory location of the current stage/suite of tests ]
CANOPUS_SHOW_RUNNING_TESTNAME      : [ 0/1 -- Allow display of testname while running in the test harness]
__PROGRAM_VARIABLE_PREFIX          : [ Prefix used for all internal variables generated by the test harness | value = 'CANOPUS' ]
